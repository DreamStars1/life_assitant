## Why

后端目前只实现了认证、用户管理、伴侣绑定和"一起做过的事"（共享记录）四个模块。前端却暴露了 Todo（待办）、Events（日程）、LifeLog（生活记录）、Push（推送通知）四个完整功能页面，这些功能在没有任何后端支持的情况下开放给用户，点击即报 502/404 错误，体验极差。

在实现相应的后端 API 之前，应先将这些未完成的功能从前端隐藏。

## What Changes

- **隐藏底部 TabBar 中的「待办」和「日历」入口**，保留「今日」「伴侣」「我的」
- **隐藏「待办」路由** — 直接访问 `/todos` 时显示"功能开发中"占位或跳转首页
- **隐藏「日程/日历」路由** — 直接访问 `/events` 时同上
- **隐藏「生活记录」入口** — 从「今日」页面的快捷"记一笔"按钮中移除「生活记录」选项，日历页面中的"实际"视图切换也一并隐藏
- **隐藏「推送通知」设置** — 设置页面中的推送开关和相关 UI 移除
- **隐藏「快捷操作」中的「加待办」按钮** — 今日页面顶部的快捷操作按钮
- **不隐藏但保持现状的内容**：认证（登录/注册/找回密码）、用户信息编辑、伴侣绑定、共享记录（"一起做过的事"）

## Capabilities

### New Capabilities

无。本次不引入新能力，仅隐藏已有 UI。所有隐藏的功能等后端实现后再重新开放。

### Modified Capabilities

无。本次不修改任何已有规约的行为，仅做 UI 隐藏。

## Impact

- **`src/pages/todos/`** — 整个待办页面目录，加占位守卫或条件渲染
- **`src/pages/events/`** — 整个日历页面目录，同上
- **`src/pages/lifelog/`** — 整个生活记录页面目录，同上
- **`src/components/TabBar.vue`** — 移除或注释「待办」和「日历」Tab
- **`src/pages/index.vue`** — 移除「加待办」按钮和生活记录快捷选项
- **`src/pages/events/index.vue`** — 屏蔽"实际"视图（LifeLog 的入口之一）
- **`src/pages/settings/index.vue`** — 移除推送通知相关 UI
- 所有隐藏均为编译时条件渲染，不删除源码，后端实现后可丝滑恢复
